## Purpose

`atmu` is an interactive, Ink-based CLI concierge for Codex CLI. The MVP goal is to reduce environment setup friction by guiding users through a single terminal UI.

## Tech Stack

- Runtime: Node.js (>= 20), ESM (`"type": "module"`)
- Language: TypeScript
- UI: React + Ink, Pastel, `@inkjs/ui`
- CLI options: Zod
- Tooling: `tsc`, Prettier, XO, AVA (`ink-testing-library` for UI tests)

## Repository Structure

- `src/`: TypeScript/TSX source
  - `src/atmu.tsx`: Pastel entry point
  - `src/commands/`: command modules (Ink-based)
  - `src/ui/`: UI components/layout/screens
- `dist/`: compiled output (generated by `tsc`)
- `test/`: AVA tests
  - UI tests render components and assert frames
  - E2E tests execute `dist/atmu.js`

## Entry Points

- Package binary: `dist/atmu.js`
- From source: build then run `node dist/atmu.js`

## Notes

- `-h/--help` prints interactive-first guidance.
- `-v/--version` prints the package version.
- Unknown CLI flags are ignored and the UI still starts.
